# Parallel Pipeline Refactoring Plan

## Overview
Refactor the FinAgent pipeline to use robust multi-threaded parallel execution with proper exception handling.

## Architecture

```
Stock Tickets (1-5) ─────────────────────────────────────────────────────────
    │
    └── ThreadPoolExecutor(max_workers=5) ─────────────────────────────────────
            │
            ├── AAPL Pipeline ────────────────────────────────────────────────
            │       │
            │       └── ThreadPoolExecutor(max_workers=7) ────────────────────
            │               │
            │               ├── Step 1: Fundamentals (parallel)
            │               ├── Step 2: Sentiment (parallel)
            │               ├── Step 3: Technical (parallel)
            │               ├── Step 4: Market (parallel)
            │               ├── Step 5: Global Economy (parallel)
            │               ├── Step 6: Fund Holdings (parallel)
            │               └── Step 7: Past Lessons (parallel)
            │                       │
            │                       └── Wait all complete
            │                               │
            │                               └── ThreadPoolExecutor(max_workers=2)
            │                                       │
            │                                       ├── Step 8.1: Bull (parallel)
            │                                       └── Step 8.2: Bear (parallel)
            │                                               │
            │                                               └── Wait both complete
            │                                                       │
            │                                                       └── Step 8.3: Debate
            │                                                               │
            │                                                               └── Step 9: Trading
            │
            ├── GOOG Pipeline (same structure)
            │
            └── MSFT Pipeline (same structure)
```

## Implementation

### 1. New `src/workflow_parallel.py`
- Replace langgraph with `concurrent.futures`
- Proper exception handling per step
- Timeout handling
- Progress tracking

### 2. Updated `finagent_api.py`
- Use new parallel workflow
- Thread-safe report generation

### 3. Exception Handling Strategy
- Each step wrapped in try/except
- Failed steps return error message but don't stop pipeline
- Logging for all failures
- Timeout per step (configurable)

## Key Design Decisions
- Use `concurrent.futures.ThreadPoolExecutor` (not ProcessPool - GIL is fine for I/O-bound)
- Each step returns tuple (result, error)
- Steps 1-7 independent, run in parallel
- Steps 8.1/8.2 depend on 1-7, run in parallel
- Steps 8.3 depends on 8.1/8.2
- Step 9 depends on 8.3
